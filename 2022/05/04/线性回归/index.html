基本形式

有的时候我们会因为各种原因去和银行贷款进行消费, 银行会根据我们个人的征信, 工资, 年龄以及其他的一些信息对我们进行评估来决定给我们的贷款数额. 这个过程就是一个典型的回归场景. 把我们的个人信息用 $x_i$ 来表示, 银行确定的贷款额度用 $y$ 来表示. 在银行评估的时候, 他们通常会从众多的个人信息项中挑选出重要的几个, 比如首先他们会考虑个人征信, 然后考虑你的薪资, 最后再考虑你的年龄. 所以每个信息项对你的贷款额度的贡献是不一样的, 用 $\theta_i$ 来表示每个信息项的权重. 则 Y 可以被表示为 $Y = x_1 \theta_{1} + x_2 \theta_{2} + …… + x_n \theta_{n}$.

线性回归是众多机器学习的预测类任务中比较简单的一个模型. 假设一个样本有d个属性, 即x = {$x_1$, $x_2$, …, $x_d$} x$\in$R, 则线性模型的目标就是去构建一个能够将输入$x_i$映射到y$\in$R的一个模型, 即

y = $w_1$$x_1$ + $w_2$$x_2$ + ... + $w_i$$x_i$ + ... +$w_d$$x_d$ + b

可以看出, 线性回归中的线性指的是因变量y和自变量x之间是线性关系, y可以用x中所有元素的加权和表示. 上式中的$w_i$就是元素$x_i$对应的权重值. b是偏置项(bias), 偏移量(offset)或者截距(intercept), 用于预测当x中所有的属性值为0时对应的预测值; 其几何意义在于对训练结果进行微调, 即对拟合面 $h_\theta(x)$ 进行上下微调 (见上图). w和b统称为模型的参数parameter. 权重w可以直观地表达出各个属性的重要性, 这也是线性模型具有很好的可解释性(comprehensibility)的原因.

环境准备

1. 构建数据集

随机构建一些数据集, 这样做的好处在于我们可以知道真实的w和b的值, 方便和学习出来的参数进行比较.

在本实验中使用w = [2, -3.4], b = 4.3来构建数据集, 其大小为num_examples = 1000

2. 构建data_iter

如前面提到的, 我们通常是一个批量一个批量地遍历数据集, 所以我们需要构建一个data_iter迭代器去随机从整个数据集中采集batch_size个样本

5. 定义损失函数

在线性回归中, 我们使用均方损失(L2 Loss).关于理由将在后面随后的文章中更新

均方损失:

$$l(w, b) = \frac{1}{2}(\hat{y} -y)^2$$
$\frac{1}{2}$的作用是为了在最后求完导数后和2约分, 简化结果.

6. 定义优化算法

这里使用小批量随机梯度下降(mini-batch stochastic gradient descent). 这也是深度学习中默认的优化算法.

  • 优化算法过程:
  1. 随机选取一组值对模型参数进行初始化;
  2. 更新参数,对参数进行迭代, 使每次迭代都可能降低损失函数的值.
    • 每次迭代中, 先随机均匀采取一个batch_size的训练数据组成小批量(mini-batch) B;
    • 然后求B中数据样本的平均损失有关模型参数的导数(梯度).
    • 最后乘以学习率(η)最为该次迭代中参数的减小量.
  • 其中B指的是批量大小(batchsize), η是指学习率(learning rate), 通常为一个比较小的正数. 他们两个是认为设定的, 因此称为超参数(hyperparameter).
