Softmax分类任务

简单介绍

分类有很多现实应用场景, 比如某个电⼦邮件是否属于垃圾邮件⽂件夹？某个⽤⼾可能注册或不注册订阅服务？某个图像描绘的是驴、狗、猫、还是鸡?等等等.

分类任务的输出通常是多个($\geq$ 2). 并且这些输入的分布很离散.所以衡量他们的大小是不太容易的. 为此分类任务中我们并不关注输出的具体值的大小, 而是偏向于置信度的比较.

softmax回归的模型本质和线性回归模型是一样的, 都是将输入x和权重w做线性加权和, 模型本质都是线性关系. 不同的地方在于输出个数从线性回归中的一个变成了标签里的类别个数.

以上图的架构为例, 其输入包含4个属性, 输出个数为3, 所以权重大小应该为(4, 3), 偏执项b的个个数为3. 输出$O_1$, $O_2$, $O_3$可以表示为:

$O_1$ = $x_1$$w_{11}$ + $x_1$$w_{21}$ + $x_3$$w_{31}$ + $x_4$$w_{41}$ + $b_1$

$O_2$ = $x_1$$w_{12}$ + $x_1$$w_{22}$ + $x_3$$w_{32}$ +$x_4$$w_{42}$ + $b_2$

$O_3$ = $x_1$$w_{13}$ + $x_1$$w_{23}$ + $x_3$$w_{33}$ +$x_4$$w_{43}$ + $b_3$


Note: 权重矩阵的大小是(输入个数, 输出个数), bias的个数为输出个数

softmax操作

在衡量分类任务的输出结果时, 这些值的范围是不确定的, 很难直观地确定比较他们的大小和含义; 其次真实标签也是一个离散的不确定范围的值. 这些导致真实值与预测值的误差不太容易衡量.

softmax操作可以同时解决这两个问题.

softmax采用了指数运算, 将所有的输出值映射到[0, 1],确保了所有输出值为非负数. 并且所有输出的和为1. (只要所有元素非负, 且和为1就可以看作是一个条件概率).

$$ \hat{y}_1, \hat{y}_2, \hat{y}_3 = softmax(O_1, O_2, O_3) $$

其中

$$\hat{y}_1 = \frac{exp(O_1)}{\sum exp(O_i)}, \hat{y}_2 = \frac{exp(O_2)}{\sum exp(O_i)}, \hat{y}_3 = \frac{exp(O_3)}{\sum exp(O_i)}$$

softmax从零实现

环境导入

1. 数据集加载

使用torchvision内置的FASHIONMNIST数据集. 该数据集包含有10个类别, 每个图像的大小为(1 * 28 * 28).

2. 构造DataLoader

torchvision下载下来的数据就是torch.dataset类型的, 所以不用像在线性回归中一样进行封装, 可以直接丢到DataLoder中构建数据迭代器

3. 定义softmax操作

softmax操作是在最后对预测值$O_i$进行转换, 所以softmax的输入形状应该是(batch_size, num_outputs). 在这里是(256, 10). 每一行就是每个样本对应的预测值$O_1$ … $O_{10}$. 对每一行进行softmax操作.

4. 初始化模型参数

权重w的形状为(num_inputs, num_outputs), 偏置b的形状为(num_outputs, 1).

5. 定义模型

模型的输入并不是一个矩阵, 而是一个4维张量(256, 1, 28, 28), 分别表示批量大小, 图像通道数和图像高和宽.官方文档中提到”A Tensor Image is a tensor with (C, H, W) shape, where C is a number of channels, H and W are image height and width. A batch of Tensor Images is a tensor of (B, C, H, W) shape, where B is a number of images in the batch.”

O = torch.matmul(X.reshape(batch_size, num_inputs), w) + b 是计算$O_i$

softmax(O)是计算$\hat{y}$
